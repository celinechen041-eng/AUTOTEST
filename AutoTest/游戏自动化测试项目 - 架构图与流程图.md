## 二、核心执行流程图

说明：流程图展示从启动项目到执行完成的全流程，包含各步骤逻辑、校验规则、重试机制。

```mermaid

flowchart TD
    %% 启动阶段
    Start[启动项目python run_game.py] --> Step1[清理旧报告（reports/目录）]
    Step1 --> Step2[读取pytest配置指定用例路径/报告目录]
    Step2 --> Step3[执行测试用例test_game_flow.py]
    
    %% 多账号批量执行
    Step3 --> Step4[读取config/user.txt获取所有测试账号]
    Step4 --> Step5{是否有未执行账号}
    Step5 -->|否| Step22[生成最终统计（成功/失败次数）]
    Step5 -->|是| Step6[取当前账号开始执行]
    
    %% HTTP登录流程
    Step6 --> Step7[调用request_sign.py发送HTTP登录请求]
    Step7 --> Step8[接收HTTP响应获取code值]
    Step8 --> Step9{HTTP code == 100?}
    Step9 -->|否| Step10[标记账号失败记录异常日志/报告]
    Step10 --> Step5[继续下一个账号]
    Step9 -->|是| Step11[提取响应中的token拼接WS地址]
    
    %% WSS连接流程
    Step11 --> Step12[调用ws_game.py尝试连接WSS（最多3次重试）]
    Step12 --> Step13{WSS连接成功?}
    Step13 -->|否| Step10[标记账号失败]
    Step13 -->|是| Step14[发送认证消息 MsgGameAuthReq]
    Step14 --> Step15[接收认证响应校验code（有则必为0）]
    Step15 --> Step16[发送进游戏消息 MsgGameEnterReq]
    Step16 --> Step17[接收进游戏响应校验code（有则必为0）]
    
    %% 50次旋转流程
    Step17 --> Step18[启动50次旋转循环]
    Step18 --> Step19[发送旋转消息 MsgGameSpinReq]
    Step19 --> Step20[接收所有旋转响应校验code（有则必为0）]
    Step20 --> Step21[记录旋转结果/赢金币统计写入日志/报告]
    Step21 --> Step18[继续下一次旋转（共50次）]
    
    %% 流程结束
    Step18 -->|50次旋转完成| Step23[标记账号成功更新统计数据]
    Step23 --> Step5[继续下一个账号]
    Step22 --> Step24[自动启动Allure报告展示全流程结果]
    Step24 --> End[流程结束]
          ```

## 三、图表说明

### 1\. 架构图核心说明

- 配置层（config/）：所有可配置项集中管理，无需修改代码，降低维护成本。

- 核心逻辑层（common/）：封装HTTP、WSS核心逻辑，供测试用例调用，实现代码复用。

- 测试用例层（test\_case/）：仅负责流程串联，不包含具体业务逻辑，便于维护。

- 输出层（reports/、logs/）：自动生成报告和日志，用于结果查看、问题排查。

### 2\. 流程图核心说明

- 全流程自动化：从启动到报告生成，无需人工干预，一键完成。

- 校验规则：严格遵循HTTP code=100、WS有code必为0的规则，异常自动拦截。

- 重试机制：WSS连接重试3次、单轮旋转重试2次，提升脚本稳定性。

- 统计功能：自动统计50次旋转的成功/失败次数、总赢金币，直观展示测试结果。
